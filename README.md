## 1. Заголовок и Источник

**Название проекта:** Оценка кредитного риска (Credit Risk Scoring)

## 2. Описание Задачи (Problem Statement)

Финансовые учреждения традиционно используют статистические модели для оценки кредитоспособности заявителей на получение кредита. Этот процесс, известный как кредитный скоринг, является критически важным для управления рисками в банковской сфере.

**Бизнес-проблема:** Необходимо разработать модель машинного обучения, которая сможет точно предсказывать вероятность дефолта (невыполнения обязательств) клиента по кредиту. На основе этой модели банк принимает решения:
- Заявителям с высоким уровнем риска отказывают в кредите
- Заявителям с умеренным риском могут предложить изменённые условия кредита (например, более высокую процентную ставку или меньшую сумму)
- Заявителям с низким уровнем риска предоставляют стандартные условия

**Важность задачи:** Эффективная оценка кредитного риска позволяет банку:
- Минимизировать финансовые потери от невозвращенных кредитов
- Оптимизировать процесс принятия решений
- Соответствовать регуляторным требованиям
- Поддерживать прибыльность кредитного портфеля

**Тип задачи:** Бинарная классификация (клиент выполнит обязательства / клиент не выполнит обязательства)

## 3. Данные (Data)

### Предоставленные файлы:

1. **`data_devsample.csv`** - основной набор данных для разработки модели
   - Содержит данные о клиентах с известным исходом кредита
   - Используется для обучения и валидации модели
   - Включает целевую переменную `TARGET`

2. **`data_to_score.csv`** - данные для оценки моделью
   - Содержит аналогичные признаки, но без целевой переменной
   - Используется для генерации прогнозов в соревновании

3. **Формат файла для отправки** (образец):
   SK_ID_CURR,SCORE


### Контекст признаков:
Данные содержат информацию о заявителях на кредит, включая:
- Демографические характеристики (возраст, пол, образование)
- Финансовые показатели (доход, сумма кредита, срок кредита)
- История кредитования (количество предыдущих кредитов, история платежей)
- Информация о занятости и другие релевантные факторы

### Целевая переменная (Target):
- **Столбец `TARGET`** - бинарная переменная, указывающая, выполнил ли клиент свои кредитные обязательства:
- `0` = клиент успешно погасил кредит ("хороший" клиент)
- `1` = клиент не выполнил обязательства ("плохой" клиент/дефолт)

## 4. Метрика Оценки (Evaluation Metric)

**Основная метрика:** Коэффициент Джини (Gini coefficient)

**Целевой показатель:** Для успешного прохождения соревнования необходимо набрать более 52 баллов по шкале Джини (что эквивалентно AUC 0.76).

### Объяснение метрики:
Коэффициент Джини - это статистическая мера, используемая для оценки качества бинарных классификаторов. В контексте кредитного скоринга:

1. **Что измеряет:** Способность модели разделять "хороших" и "плохих" клиентов
2. **Интерпретация:**
- 0% (0.0) - модель не лучше случайного угадывания
- 100% (1.0) - идеальное разделение классов
3. **Связь с AUC:** Коэффициент Джини = 2 × AUC - 1
- Целевой показатель 52% по Джини соответствует AUC 0.76
4. **Бизнес-значение:** Более высокая метрика Джини означает, что модель лучше ранжирует клиентов по риску, что позволяет банку более точно принимать решения о кредитовании.
